# Medical-Store-Management

💊 Medical Store Management System (SQL Project)

📖 Description

This project is a relational database system designed to manage the operations of a medical store efficiently. It handles data related to medicines, companies, dealers, employees, customers, sales, purchases, stock, and payments.
The system is built using SQL (MySQL) and demonstrates real-world database concepts like relationships, constraints, and normalization.

It helps in:

Tracking medicine inventory 📦
Managing customer transactions 💳
Monitoring purchases & sales 📊
Maintaining employee and dealer records 👨‍💼

🛠️ Tech Stack
💾 Database: MySQL
🧠 Language: SQL
🗂️ Concepts Used:
Primary & Foreign Keys
Joins
Constraints (CHECK, NOT NULL)
Auto Increment
Relational Database Design 

✨ Features
🏢 Company Management
Store company details like name, location, and contact
Link medicines and dealers to companies
💊 Medicine Management
Maintain medicine details with price, manufacture date, expiry date
Track which company produces each medicine
🚚 Dealer Management
Store dealer information and company association
Manage supply chain efficiently
👨‍💼 Employee & Login System
Store employee roles (Manager, Cashier, etc.)
Secure login system linked with employees
🧑‍🤝‍🧑 Customer Management
Maintain customer records
Track purchase behavior
🛒 Sales Management
Record medicine sales
Calculate total price automatically
Track customer purchases
📦 Purchase Management
Record stock purchases from dealers
Maintain supplier transactions
📊 Stock Management
Track available quantity of medicines
Prevent negative stock using constraints
💳 Payment System
Record customer payments
Support multiple payment modes (Cash, UPI, Card)
⭐ Regular Customer Benefits
Identify loyal customers
Apply discounts based on purchase count
